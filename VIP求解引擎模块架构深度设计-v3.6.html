3. 多目标转单目标数学推导

3.1 原始多目标问题定义

maximize: f₁(x) = VIP_rate(x) # 目标 1：最大化 VIP 速率 maximize: f₂(x) = Common_rate(x) # 目标 2：最大化普通用户速率 subject to: x ∈ X # 可行域（模块组合 × 参数值） 其中： x = {x_BEGBR, x_FWA, x_DACQ, x_RRC} ∈ {0, 0.25, 0.5, 0.75, 1.0} # 参数档位 X = 所有可能的模块组合和参数值构成的有限离散空间

3.2 ε-约束法转化（推荐）

转化为单目标+约束问题： maximize: f₁(x) = VIP_rate(x) ← 单目标 subject to: f₂(x) >= ε ← 普通用户底线约束 x ∈ X ← 可行域 ε 的扫描策略： ε ∈ [min(Common_rate), max(Common_rate)] 扫描步长：Δε = (max - min) / N, N = 10-20 求解步骤： 1. 设定 ε₁ = min(Common_rate) 2. 求解 maximize f₁(x) s.t. f₂(x) >= ε₁ → 得到 Pareto 解 p₁ 3. ε₂ = ε₁ + Δε, 求解 → p₂ 4. 重复直到 ε = max(Common_rate) 5. 合并所有 p₁, p₂, ..., pN → 完整 Pareto 前沿

3.3 Pareto 前沿选择策略

得到完整 Pareto 前沿后，如何选择最优解？

选择规则数学表达适用场景
VIP 优先保障推荐选择 VIP_rate 最高的 Pareto 解，同时确保 Common_rate >= 底线VIP 保障场景
底线保护如果 VIP 提升导致 Common 低于底线，选择最接近底线的 Pareto 解高负载场景
均衡选择加权评分 = α × VIP_score + (1-α) × Common_score, α = 0.7需要平衡的场景

3.4 其他转化方法对比

方法数学形式优点缺点适用场景
ε-约束法推荐max f₁, s.t. f₂>=ε数学全局最优大规模慢离散小规模（当前场景）

4.2 为什么推荐 ε-约束枚举法？

选择理由

  1. 问题规模小：4 模块 × 5 档位 = 625 种组合（可完全枚举）
  2. 需要最优性证明：启发式算法无法证明找到的是全局最优
  3. 离散问题：参数是离散档位（0/0.25/0.5/0.75/1.0），不适合连续优化算法
  4. 数学保证：最优性间隙 = 0.0000，可严格证明
  5. 实现简单：无需复杂进化操作，易于理解和验证

知识压缩后：625 → ~20 种，枚举速度 < 1ms

4.3 大规模网络扩展方案

当小区数量 > 100 时的扩展策略：

方案原理适用场景
方案 1：小区级并行 + 独立求解每个小区独立求解（解空间互不影响），并行执行，总时间 = max(单个小区求解时间)小区间无耦合
方案 2：增量求解只重新求解状态变化的小区，其他小区复用上次最优解，减少 80%+ 求解量状态缓慢变化
方案 3：区域级分组求解将相邻小区分组（同 PRB 资源池），组内联合优化，组间独立，平衡求解质量和速度小区间有耦合

5.3 核心模块详细设计

5.3.1 StrategyPreFilter（策略预过滤器）

职责：根据现有代码规则，过滤非法策略组合，确保求解器只在合法策略空间搜索。

核心逻辑

条件策略选择
LTE 初始规划BEGBR + DACQ（提+限），用户数 > 最大可保障用户数
NR 初始规划FWA（仅提）
后续调整（LTE）可组合使用 BEGBR + DACQ，遵循"提>限>拒"优先级
后续调整（NR）可组合使用 FWA + RRC，遵循"提>限>拒"优先级
class StrategyPreFilter: def filter_valid_strategies(self, cell: CellState) -> List[List[str]]: """根据 RAT 类型和初始规划状态，返回合法策略组合""" if cell.rat == "LTE": if cell.is_first_time_setup: return [["BEGBR", "DACQ"]] # LTE 初始规划 else: return [["BEGBR"], ["DACQ"], ["BEGBR", "DACQ"]] # 后续调整 elif cell.rat == "NR": if cell.is_first_time_setup: return [["FWA"]] # NR 初始规划 else: return [["FWA"], ["RRC"], ["FWA", "RRC"]] # 后续调整 return []

5.3.2 DynamicParamBounds（动态参数边界）

职责：根据 VIP 目标速率动态计算各模块参数边界，替代固定值。

class DynamicParamBounds: def calculate_begbr_bounds(self, cell): """BEGBR 边界：left=VIP LTE 目标, right=2倍目标""" target_kbps = cell.vip_target_rate * 1000 return { "left_bound": target_kbps, "right_bound": target_kbps * 2, "default": target_kbps } def calculate_fwa_bounds(self, cell): """FWA 边界：left=VIP NR 目标, right=2倍目标""" target_kbps = cell.vip_target_rate * 1000 return { "left_bound": target_kbps, "right_bound": target_kbps * 2, "default": target_kbps } def calculate_dacq_bounds(self): """DACQ 边界：固定值 3000000-5000000""" return {"left_bound": 3000000, "right_bound": 5000000, "default": 5000000} def calculate_rrc_bounds(self): """RRC 边界：固定值 10-100""" return {"left_bound": 10, "right_bound": 100, "default": 100} def generate_discrete_levels(self, left_bound, right_bound, levels=5): """生成离散档位""" step = (right_bound - left_bound) / (levels - 1) return [round(left_bound + i * step) for i in range(levels)]

5.3.3 ProgressiveCompressor（渐进式压缩器）

职责：根据差距比例压缩解空间，对齐现有代码的渐进式调整逻辑。

压缩规则

场景压缩策略解空间大小
初次规划只枚举默认值档位1 种组合
差距 > 30%枚举所有 5 档位5^n 种组合
差距 < 30%只枚举相邻档位（±1 档）3^n 种组合
class ProgressiveCompressor: def compress_solution_space(self, cell, current_params, gap_percentage): """根据差距比例压缩解空间""" if cell.is_first_time_setup: return self._get_default_params(cell) elif gap_percentage > 30: return self._enumerate_all_levels(cell) else: return self._enumerate_adjacent_levels(cell, current_params) def _get_default_params(self, cell): """初次规划 - 只使用默认值""" bounds = DynamicParamBounds().get_all_bounds(cell) return {module: bounds[module]["default"] for module in cell.modules} def _enumerate_all_levels(self, cell): """差距 > 30% - 枚举所有档位""" bounds = DynamicParamBounds().get_all_bounds(cell) levels = {} for module in cell.modules: levels[module] = DynamicParamBounds().generate_discrete_levels( bounds[module]["left_bound"], bounds[module]["right_bound"], 5 ) return levels def _enumerate_adjacent_levels(self, cell, current_params): """差距 < 30% - 只枚举相邻档位（±1）""" bounds = DynamicParamBounds().get_all_bounds(cell) adjacent_params = {} for module, current_value in current_params.items(): all_levels = DynamicParamBounds().generate_discrete_levels( bounds[module]["left_bound"], bounds[module]["right_bound"], 5 ) current_level_idx = self._find_closest_level(all_levels, current_value) adjacent_indices = [ max(0, current_level_idx - 1), current_level_idx, min(4, current_level_idx + 1) ] adjacent_params[module] = [all_levels[i] for i in adjacent_indices] return adjacent_params

5.3.4 GainModel（增益预测模型）

职责：预测策略调整后的 VIP 和普通用户速率增益。

class GainModel: def predict_vip_gain(self, cell, module, param_value): """预测 VIP 速率增益（基于经验公式）""" if module == "BEGBR": # BEGBR 提升：VIP 速率提升约 20%-40% return cell.vip_curr_rate * (1.2 + 0.2 * (param_value - cell.vip_target_rate * 1000) / 1000) elif module == "FWA": # FWA 提升：VIP 速率提升约 15%-35% return cell.vip_curr_rate * (1.15 + 0.2 * (param_value - cell.vip_target_rate * 1000) / 1000) elif module == "DACQ": # DACQ 限制：间接提升 VIP 速率 5%-15% return cell.vip_curr_rate * 1.1 return cell.vip_curr_rate def predict_common_impact(self, cell, module, param_value): """预测普通用户速率影响""" if module in ["BEGBR", "FWA"]: # 提升类策略：普通用户速率下降 5%-15% return cell.common_curr_rate * 0.9 elif module == "DACQ": # 限制类策略：普通用户速率下降 10%-25% return cell.common_curr_rate * 0.85 elif module == "RRC": # RRC 限制：普通用户速率下降 15%-30% return cell.common_curr_rate * 0.8 return cell.common_curr_rate

5.3.6 CommonRateEstimator（普通用户指标估算）

职责：补全现有代码缺失的普通用户速率获取路径。

class CommonRateEstimator: def estimate_common_rate(self, cell): """估算普通用户速率（基于 PRB 反推）""" # 估算公式： # Common_rate = (Total_PRB_capacity - VIP_rate * VIP_users) / Common_users prb_total = 100 # 假设 PRB 总量 spectral_efficiency = 5.0 # bps/Hz/PRB（经验值） total_capacity = cell.prb_load * prb_total * spectral_efficiency vip_consumption = cell.vip_curr_rate * cell.vip_user_count estimated_common_users = cell.estimated_common_user_count if estimated_common_users > 0: common_rate = (total_capacity - vip_consumption) / estimated_common_users return max(0, common_rate) else: return 0.0 def calculate_common_min_rate(self, cell): """计算普通用户最低可接受速率""" return cell.vip_target_rate * 0.3 # 经验公式：VIP 目标的 30%

5.3.7 EvaluationAligner（评估对齐器）

职责：求解器输出增加 20% 阈值评估字段，与现有代码评估规则对齐。

class EvaluationAligner: def calculate_evaluate_result(self, current_rate, target_rate): """计算评估结果 - 与现有代码完全对齐""" percentage_diff = abs(current_rate - target_rate) / target_rate percentage_diff = round(percentage_diff, 3) if percentage_diff <= 0.2: return "AchievedTarget" elif current_rate > target_rate: return "ExceedTarget" else: return "NotAchievedTarget" def align_result(self, result: OptimizationResult) -> OptimizationResult: """对齐求解器输出，增加评估字段""" result.vip_percentage_diff = abs(result.vip_predicted - result.cell.vip_tgt_rate) / result.cell.vip_tgt_rate result.vip_evaluate_result = self.calculate_evaluate_result(result.vip_predicted, result.cell.vip_tgt_rate) result.common_percentage_diff = abs(result.common_predicted - result.cell.common_min_rate) / result.cell.common_min_rate result.common_evaluate_result = self.calculate_evaluate_result(result.common_predicted, result.cell.common_min_rate) result.needs_adjustment = result.vip_evaluate_result != "AchievedTarget" if result.vip_evaluate_result == "ExceedTarget": result.adjustment_direction = "decrease" elif result.vip_evaluate_result == "NotAchievedTarget": result.adjustment_direction = "increase" else: result.adjustment_direction = "none" return result

class DynamicParamBounds: """动态参数边界计算器""" def calculate_begbr_bounds(self, cell): """BEGBR 参数边界动态计算""" target_rate_kbps = cell.vip_target_rate * 1000 # Mbps → kbps return { "left_bound": target_rate_kbps, "right_bound": target_rate_kbps * 2, "default": target_rate_kbps } def calculate_fwa_bounds(self, cell): """FWA 参数边界动态计算""" target_rate_kbps = cell.vip_target_rate * 1000 return { "left_bound": target_rate_kbps, "right_bound": target_rate_kbps * 2, "default": target_rate_kbps } def calculate_dacq_bounds(self): """DACQ 参数边界（固定值）""" return {"left_bound": 3000000, "right_bound": 5000000, "default": 5000000} def calculate_rrc_bounds(self): """RRC 参数边界（固定值）""" return {"left_bound": 10, "right_bound": 100, "default": 100} def generate_discrete_levels(self, left_bound, right_bound, levels=5): """生成离散参数档位""" step = (right_bound - left_bound) / (levels - 1) return [round(left_bound + i * step) for i in range(levels)] def get_all_bounds(self, cell): """获取所有模块的参数边界""" bounds = {} if cell.rat == "LTE" or "BEGBR" in cell.modules: bounds["BEGBR"] = self.calculate_begbr_bounds(cell) if cell.rat == "NR" or "FWA" in cell.modules: bounds["FWA"] = self.calculate_fwa_bounds(cell) if "DACQ" in cell.modules: bounds["DACQ"] = self.calculate_dacq_bounds() if "RRC" in cell.modules: bounds["RRC"] = self.calculate_rrc_bounds() return bounds

6.7 与求解器融合

融合架构

将策略选择逻辑和参数边界处理策略与求解器深度融合，确保求解器在合法策略空间和参数边界内搜索。

class SolverIntegrator: def integrate_solver_with_strategy(self, cell: CellState, solver: MultiObjectiveOptimizer): """将求解器与策略选择逻辑和参数边界处理策略融合""" # 1. 策略预过滤 valid_strategies = StrategySelector().select_initial_strategy(cell) # 2. 动态参数边界计算 bounds = DynamicParamBounds().get_all_bounds(cell) # 3. 解空间压缩 compressed_space = ProgressiveCompressor().compress_solution_space(cell, bounds) # 4. 调用求解器 result = solver.solve(cell, valid_strategies, compressed_space) # 5. 边界验证 if not self._validate_bounds(result, bounds): # 边界验证失败，使用默认值回退 result = self._fallback_to_default(result, bounds) # 6. 评估对齐 result = EvaluationAligner().align_result(result) return result def _validate_bounds(self, result, bounds): """验证求解结果是否在参数边界内""" for module, param_value in result.params.items(): if module in bounds: if param_value < bounds[module]["left_bound"] or param_value > bounds[module]["right_bound"]: return False return True def _fallback_to_default(self, result, bounds): """边界验证失败时的默认值回退""" for module in result.params: if module in bounds: result.params[module] = bounds[module]["default"] return result

6.8 评估规则集成 - 20% 阈值对齐

评估规则对齐

将求解器输出与现有代码的评估规则（20% 阈值）对齐，确保求解器输出可以直接用于 AgentLoop 的决策逻辑。

class EvaluationAligner: def calculate_evaluate_result(self, current_rate, target_rate): """计算评估结果 - 与现有代码完全对齐""" percentage_diff = abs(current_rate - target_rate) / target_rate percentage_diff = round(percentage_diff, 3) if percentage_diff <= 0.2: return "AchievedTarget" elif current_rate > target_rate: return "ExceedTarget" else: return "NotAchievedTarget" def align_result(self, result: OptimizationResult) -> OptimizationResult: """对齐求解器输出，增加评估字段""" result.vip_percentage_diff = abs(result.vip_predicted - result.cell.vip_tgt_rate) / result.cell.vip_tgt_rate result.vip_evaluate_result = self.calculate_evaluate_result(result.vip_predicted, result.cell.vip_tgt_rate) result.common_percentage_diff = abs(result.common_predicted - result.cell.common_min_rate) / result.cell.common_min_rate result.common_evaluate_result = self.calculate_evaluate_result(result.common_predicted, result.cell.common_min_rate) result.needs_adjustment = result.vip_evaluate_result != "AchievedTarget" if result.vip_evaluate_result == "ExceedTarget": result.adjustment_direction = "decrease" elif result.vip_evaluate_result == "NotAchievedTarget": result.adjustment_direction = "increase" else: result.adjustment_direction = "none" return result

7. 渐进式调整压缩器

7.1 现有代码的参数调整规则

# 现有代码的参数调整逻辑（policies-build SKILL.md） IF first_time_setup: → 使用默认值（left_bound） ELSE IF gap > 30%: → 激进调整（二分法） ELSE IF gap < 30%: → 缓慢调整（固定步长 = 取值范围的 10%-30%）

7.2 渐进式压缩器设计

将渐进式调整逻辑映射到求解器的解空间压缩阶段：

初次规划
→
只枚举默认值档位
→
解空间 = 1 种组合
差距 > 30%
→
枚举所有 5 档位
→
解空间 = 5^n 种组合
差距 < 30%
→
只枚举相邻档位（±1 档）
→
解空间 = 3^n 种组合

7.3 代码实现

class ProgressiveAdjustmentCompressor: """渐进式调整压缩器""" def compress_solution_space(self, cell, current_params, gap_percentage): """根据差距比例，压缩参数枚举空间""" if cell.is_first_time_setup: return self._get_default_params(cell) elif gap_percentage > 30: return self._enumerate_all_levels(cell) else: return self._enumerate_adjacent_levels(cell, current_params) def _get_default_params(self, cell): """初次规划 - 只使用默认值""" bounds = DynamicParamBounds().get_all_bounds(cell) return {module: bounds[module]["default"] for module in cell.modules} def _enumerate_all_levels(self, cell): """差距 > 30% - 枚举所有档位""" bounds = DynamicParamBounds().get_all_bounds(cell) levels = {} for module in cell.modules: levels[module] = DynamicParamBounds().generate_discrete_levels( bounds[module]["left_bound"], bounds[module]["right_bound"], 5 ) return levels def _enumerate_adjacent_levels(self, cell, current_params): """差距 < 30% - 只枚举相邻档位（±1）""" bounds = DynamicParamBounds().get_all_bounds(cell) adjacent_params = {} for module, current_value in current_params.items(): all_levels = DynamicParamBounds().generate_discrete_levels( bounds[module]["left_bound"], bounds[module]["right_bound"], 5 ) current_level_idx = self._find_closest_level(all_levels, current_value) adjacent_indices = [ max(0, current_level_idx - 1), current_level_idx, min(4, current_level_idx + 1) ] adjacent_params[module] = [all_levels[i] for i in adjacent_indices] return adjacent_params def _find_closest_level(self, levels, value): """找到最接近当前值的档位索引""" min_diff = float('inf') closest_idx = 0 for i, level in enumerate(levels): diff = abs(level - value) if diff < min_diff: min_diff = diff closest_idx = i return closest_idx

8.3 路径 1 实现（短期推荐）

class CommonRateEstimator: """普通用户速率估算器""" def estimate_common_rate(self, cell): """ 估算公式： Common_rate = (Total_PRB_capacity - VIP_rate * VIP_users) / Common_users 其中： - Total_PRB_capacity = PRB_load * PRB_total * spectral_efficiency - VIP_rate = cell.vip_curr_rate - VIP_users = cell.vip_user_count - Common_users = 估算值（基于历史平均值） """ prb_total = 100 # 假设 PRB 总量 spectral_efficiency = 5.0 # bps/Hz/PRB（经验值） total_capacity = cell.prb_load * prb_total * spectral_efficiency vip_consumption = cell.vip_curr_rate * cell.vip_user_count estimated_common_users = cell.estimated_common_user_count if estimated_common_users > 0: common_rate = (total_capacity - vip_consumption) / estimated_common_users return max(0, common_rate) else: return 0.0 def calculate_common_min_rate(self, cell): """计算普通用户最低可接受速率""" return cell.vip_target_rate * 0.3 # 经验公式
