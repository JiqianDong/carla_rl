1. 问题定义与目标分析

1.1 业务场景

在运营商无线网络中，小区资源（PRB、连接数等）有限。需要通过MML策略特性对VIP用户和普通用户进行差异化速率保障：

  • VIP用户: 保障速率要求较高，优先级最高
  • 普通用户: 保障速率要求较低，优先级低于VIP，但不得低于最低底线

1.2 可用策略模块

模块RAT作用VIP影响普通用户影响强度
BEGBRLTEVIP调度优先级提升正向轻微负向温和
FWANRVIP带宽分配提升正向轻微负向温和
DACQLTE普通用户AMBR限速间接正向强负向中等
RRCNR普通用户接入数限制间接正向严重负向强烈

1.3 目标冲突分析

资源总量有限 → VIP提升 = 普通用户下降 (零和博弈)

目标1: max(VIP速率) → 需要更多资源
目标2: min(普通用户速率下降) → 需要保留资源

极端场景: 即使VIP保障速率也不达标 → 只能通过降低普通用户速率目标来优先保障VIP

1.4 待解决的三个关键问题

问题挑战本方案对策

3. 关键算法设计

3.1 问题形式化

决策变量: x = {x_BEGBR, x_FWA, x_DACQ, x_RRC} ∈ [0, 1]

目标函数:
f1(x) = max(VIP速率) = VIP_current + G_vip(x) # 最大化VIP速率
f2(x) = min(普通用户损失) = Common_current + G_common(x) # 最小化普通用户损失

约束条件:
g1: Common_predicted(x) ≥ Common_min_rate # 硬约束
g2: x_i ∈ {0, 0.25, 0.5, 0.75, 1.0} # 参数档位
g3: x_i = 0 if module_i not available for RAT # RAT绑定

3.2 ε-约束精确求解算法

转换: max f1(x) s.t. f2(x) ≥ epsilon, g1, g2, g3

流程:
1. 生成ε网格: epsilon ∈ [Common_min, Common_current], 10档
2. 对每个ε，枚举所有参数组合，找满足约束的最大VIP解
3. 收集所有ε值最优解，过滤支配解
4. 输出Pareto前沿

3.3 增益预测模型

G_vip(x) = Σ [base_gain_i × x_i × load_sensitivity × marginal_effect_i]
load_sensitivity = max(1.0, 1.0 + 3.0 × (PRB - 0.7))
marginal_effect_i = max(0.4, 1.0 - 0.3 × i)
模块base_vip_gainbase_common_loss说明
BEGBR0.15-0.05温和策略
FWA0.18-0.06略高于4G
DACQ0.25-0.20中等影响
RRC0.40-0.35影响最大

4. 最优性证明方法

4.1 证明思路

由于问题是有限离散优化问题（参数档位有限），通过完全枚举 + Pareto支配验证证明最优性。

4.2 证明步骤

  1. 解空间界定: 明确所有可能的参数组合（有限集）
  2. 约束过滤: 移除违反硬约束的候选解
  3. 完全评估: 对剩余候选解逐一计算目标函数值
  4. Pareto过滤: 移除所有被支配的解
  5. 最优性间隙: 比较所选解与Pareto前沿的差距

4.3 最优性间隙

optimality_gap = (max_VIP_in_Pareto - selected_VIP) / max_VIP_in_Pareto
如果 < 0.01 → 数学上保证全局最优

8. 算法实现与代码

8.1 核心求解器

class MultiObjectiveOptimizer:
    """多目标量化优化引擎"""
    
    def __init__(self):
        self.gain_model = GainModel()
        self.epsilon_solver = EpsilonConstraintSolver(self.gain_model)
        self.compressor = KnowledgeDrivenCompressor(self.gain_model)
        self.pareto_merger = ParetoFrontMerger()
        self.interpreter = InterpretabilityPipeline()
    
    def solve(self, cell: CellState) -> OptimizationResult:
        # Step 1: 解空间压缩
        modules, levels, stats = self.compressor.compress(cell)
        # Step 2: ε-约束求解
        pareto_front = self.epsilon_solver.solve(cell)
        # Step 3: 枚举验证（保证最优性）
        all_candidates = self._enumerate_all(cell, modules, levels)
        all_pareto = self.pareto_merger.merge([pareto_front, all_candidates])
        # Step 4: 选择最优解（VIP优先）
        best = self._select_best(cell, all_pareto)
        # Step 5: 计算最优性间隙
        gap = self._calc_optimality_gap(cell, best, all_pareto)
        # Step 6: 生成解释
        result = OptimizationResult(cell=cell, best_candidate=best,
            pareto_front=all_pareto, optimality_gap=gap, ...)
        result.interpretation = self.interpreter.generate_interpretation(cell, result)
        return result

8.2 ε-约束求解器

class EpsilonConstraintSolver:
    def solve(self, cell, epsilon_grid=None):
        if epsilon_grid is None:
            epsilon_grid = np.linspace(cell.common_min_rate, cell.common_current_rate, 10)
        solutions = []
        for epsilon in epsilon_grid:
            best = self._solve_single_epsilon(cell, epsilon)
            if best:
                solutions.append(best)
        return self._filter_pareto_front(solutions)
    
    def _solve_single_epsilon(self, cell, epsilon):
        best = None
        for params in product(param_levels, repeat=len(modules)):
            pred = self.gain_model.predict(cell, params)
            if pred["common_predicted"] >= epsilon:
                if pred["vip_predicted"] > best_vip:
                    best = pred
        return best

8.3 Agent集成

class QuantPlanBridge:
    def plan(self, cells):
        optimizer = MultiObjectiveOptimizer()
        results = []
        for cell in cells:
            result = optimizer.solve(cell)
            results.append({
                "site_name": cell.site_name,
                "modules": result.best_candidate.modules,
                "vip_predicted": result.best_candidate.vip_predicted_rate,
                "common_predicted": result.best_candidate.common_predicted_rate,
                "optimality_gap": result.optimality_gap,
                "interpretation": result.interpretation,
            })
        return results

10. Skill Prompt 改造方案

10.1 evaluate Skill 改造

- 仅查询VIP用户指标 + 查询BOTH VIP和common用户指标 + 使用QueryThroughputRateCommon查询普通用户 + 输出格式增加: common_evaluate_result, common_curr_rate, common_min_rate + 普通用户默认阈值: common_min_rate = vip_target * 0.3

10.2 polices-plan Skill 改造（核心）

- Step 3: 应用专家经验 → Step 4: 规则匹配 + Step 3: 调用量化求解器 quant_optimizer_solve (Priority 1) + 构建 cell_states 数组 → 调用工具 → 解析结果 + 如果求解成功 → 直接使用 best_modules 和 best_params + 如果求解失败 → 降级到 Step 4 规则匹配 + 输出增加 "source" 字段: "quant_optimizer" 或 "rule_fallback"

10.3 policies-build Skill 改造

+ 新增: 当 source=="quant_optimizer" 时 + 直接使用求解器输出的参数值 + 校验 left_bound / right_bound + 超出边界则裁剪并记录警告 + 保持: 原有规则计算逻辑作为降级

10.4 policies-effect-predict Skill 改造

- 仅评估VIP状态: AchievedTarget/NotAchievedTarget/ExceedTarget + 双目标评估: VIP状态 + Common状态 + ISSUABLE条件: 所有小区VIP达标 AND 所有小区普通用户达标 + 普通用户不达标调整: 放松DACQ/RRC限制
